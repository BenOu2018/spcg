# SPCG 健康游戏化投入系统

> 目标不是制造沉迷，而是让学生愿意持续学习算法、愿意挑战、愿意修错，并在系统层面避免过度使用、焦虑驱动和黑暗模式。

## 1. Product Position

SPCG 的游戏化系统命名为“健康投入系统”或“学习留存系统”，不以“沉迷系统”为产品目标。

当前可复用的系统底座：

- 地图与关卡推进
- 金币、蒜粒、段位、称谓、背包
- 段位赛与考试 attempt
- 老师端学生管理
- 提交记录、进度记录、奖励账本

后续设计应围绕“学习质量”而不是“在线时长”优化。

## 2. Design Principles

- 让学生爱上算法学习，而不是爱上无限刷奖励。
- 奖励真实能力成长：AC、修错、复习、考试、知识点覆盖。
- 不奖励挂机、无意义在线、机械重复提交。
- 不做强制连续登录，不做断签惩罚，不做深夜限时活动。
- 不用高压排行榜作为低龄学生的核心驱动力。
- 所有金币、蒜粒、物品、称谓来源必须可解释、可审计、有上限。
- 未成年人相关功能必须考虑休息提醒、时间边界、家长或老师可见性。

## 3. Game Mechanisms To Learn And Adapt

| 游戏机制 | SPCG 改造方式 | 设计边界 |
|---|---|---|
| 每日任务 | 改为短时学习目标，例如 1 道复习题、1 次修错、1 个知识点回顾 | 不做必须每天登录，不做断签惩罚 |
| 连续登录 | 改为学习节奏，例如本周学习 3 天、完成 3 次短练 | 允许断签保护，强调恢复节奏 |
| 随机奖励 | 改为可解释掉落，例如隐藏蒜粒、背包物品 | 有概率说明、次数上限、账本记录 |
| 排行榜 | 改为小组成长榜、自我进步榜、知识点完成榜 | 不按在线时长排名，不制造羞辱感 |
| 地图推进 | 依赖真实 AC、考试、复习完成 | 不允许挂机推进 |
| 段位称谓 | 奖励能力、考试、隐藏挑战 | 不因单纯登录或停留时长提升 |
| 商城 | 优先兑换外观、称谓、收藏物 | 不出售影响判题能力的优势 |
| 社交 | 老师/学生、班级内互相鼓励 | 不做陌生人压力社交和攀比消费 |

## 4. White / Gray / Black List

### White List

- 地图阶段奖励
- 首次 AC 奖励
- 修错奖励
- 周目标奖励
- 复习完成奖励
- 考试通过奖励
- 自我进步反馈
- 老师可见学习报告
- 休息提醒和学习节奏提示

### Gray List

这些机制可以使用，但必须加边界：

- 隐藏掉落：必须有上限和账本。
- 排行榜：只能用于小组、阶段、自我进步，不做全站无差别排名。
- 商城：只兑换外观、称谓、背包展示，不影响判题或考试。
- 活动任务：不能依赖深夜时段，不能用错过惩罚驱动。
- AI 个性化推荐：只推荐学习路径，不推断敏感人格或智力标签。

### Black List

- 强制连续登录奖励
- 断签清零或惩罚
- 无限刷金币、无限刷蒜粒
- 深夜限时奖励
- 按在线时长发主奖励
- 高压全站排名
- 面向未成年人的诱导消费
- 自动推断性别、智力、EQ 等敏感标签

## 5. Metrics

### Learning Quality Metrics

- AC 题数
- 首次 AC 率
- 修错次数
- WA 后再次提交成功率
- 知识点覆盖率
- 复习完成率
- 考试通过率
- 阶段地图完成率

### Healthy Usage Metrics

- 单次学习时长
- 连续学习时长
- 深夜使用次数
- 短时间重复无效提交次数
- 失败后退出率
- 休息提醒触达率
- 老师/家长查看学习报告次数

### Metrics Not To Optimize Directly

- 总在线时长
- 每日打开次数
- 连续登录天数
- 深夜活跃
- 无意义点击量

## 6. Anti-Addiction And Safety Boundaries

第一版建议先做柔性健康边界：

- 单次学习满 30 分钟提醒休息。
- 连续学习满 60 分钟强提示休息，不再突出奖励动画。
- 夜间时段弱化任务入口，不推送限时奖励。
- 每日建议目标达到后，提示“今天主线学习已完成，可以休息或轻量复习”。
- 重复无效提交过多时，提醒查看提示、题解或请老师帮助。

面向未成年人正式运营前，需要做合规评估：

- 如果产品按网络游戏形态运营，需要评估实名、时段、时长、防沉迷要求。
- 如果产品按教育学习产品运营，也应保留时间提醒、监护、隐私保护和数据最小化原则。
- 合规规则可能变化，正式上线前必须按最新法律法规复核。

合规参考：

- [中国政府网：国家新闻出版署关于进一步严格管理切实防止未成年人沉迷网络游戏的通知](https://www.gov.cn/zhengce/zhengceku/2021-09/01/content_5634661.htm)

## 7. V1 Implementation Suggestion

V1 只做小闭环，不做复杂 AI 和重度行为画像：

1. 每日学习目标：推荐 1 道新题、1 道复习题或 1 次修错。
2. 周目标：按知识点覆盖和修错完成结算。
3. 地图阶段奖励：每完成一小段地图，发放金币、称谓或背包展示物。
4. 健康提醒：学习 30 分钟提醒休息，60 分钟强提示。
5. 老师视图：展示学习质量指标，不展示羞辱式排名。
6. 奖励审计：所有奖励继续写入 ledger，避免重复刷。

## 8. SPCG Integration Plan

健康投入系统要融入现有 SPCG，而不是另起一套玩法。核心原则是：

> 地图给方向，题目给挑战，反馈给信心，奖励给记忆，教练给下一步，健康系统负责拉住边界。

### 8.1 地图和首页

地图页要让学生一打开就知道“今天做什么”。

- 显示今日学习目标：1 道新题、1 道修错题、1 个知识点复习三选一。
- 今日目标不要求连续登录，不做断签惩罚。
- 地图节点状态除 locked/current/completed 外，后续可增加 review/retry/challenge 状态。
- 完成一小段地图后给阶段奖励，例如金币、称谓进度或背包展示物。
- 地图推进必须依赖真实 AC、考试通过或复习完成，不允许挂机推进。

### 8.2 编程关卡

关卡页要把失败转化为“我还能继续”的信号。

- WA 后优先提示下一步，而不是只显示失败。
- 连续 WA 2 次后推荐查看第 1 个提示或切到 A 线练习。
- 连续 WA 3 次后推荐题解视频、同知识点例题或向老师求助。
- AC 后展示本题成长反馈，例如修错次数、掌握知识点、段位/金币变化。
- 修错成功可以给少量奖励或背包碎片，但不能超过首次 AC 主奖励。

### 8.3 A/B 线

A/B 线用于适配不同水平学生。

- A 线是主线简单路径，保证学生持续推进。
- B 线是挑战路径，奖励隐藏蒜粒、稀有称谓、背包物品或阶段徽章。
- 学生卡住时推荐回 A 线补基础。
- 学生表现稳定时推荐 B 线挑战。
- A/B 线都必须绑定同一课程大纲，不能为了刺激感引入跳跃式难度。

### 8.4 奖励系统

奖励只奖励有效学习行为。

- 首次 AC 发金币。
- 修错成功发少量修错奖励。
- 阶段考试通过发蒜粒。
- B 线挑战成功发稀有称谓或背包物品。
- 重复刷同一题不重复发主奖励。
- 所有奖励都必须写入 ledger，并保留来源、上限和唯一性约束。

### 8.5 Me 页面

`/me` 是成长反馈面板，不只是提交记录页。

前 4 个核心数据优先展示：

- 今日目标完成情况。
- 当前段位，以及距离下一级还差多少。
- 本周掌握知识点。
- 最近修错成功次数。

菜单展开区展示：

- 背包。
- 称谓。
- 奖励记录。
- 做过的题。
- 考试记录。

### 8.6 教练系统

教练系统第一版只给一个明确下一步，不一次塞满任务。

- 今天建议修正哪一道题。
- 今天建议复习哪个知识点。
- 是否适合挑战 B 线。
- 是否已经完成主线目标，可以休息或轻量复习。
- 老师端看到的是学习质量和薄弱点，不是在线时长排行。

### 8.7 健康提醒

健康提醒必须和奖励系统联动。

- 学习 30 分钟提醒休息。
- 学习 60 分钟强提示休息。
- 夜间不推送限时任务和高刺激奖励。
- 每日目标完成后，提示“今天主线学习已完成，可以休息或轻量复习”。
- 超过建议学习量后，降低奖励动画强度，不再制造继续刷的压力。

## 9. Minimum Feature Set For Upcoming Tasks

后续任务优先按这个最小闭环实施：

1. 今日学习目标。
2. 修错奖励和修错反馈。
3. A/B 线推荐。
4. `/me` 成长反馈面板。
5. 30/60 分钟健康提醒。

这些功能完成后，再考虑周目标、阶段奖励、老师端学习建议、背包展示和个性化推荐。

## 10. Review Questions For New Features

每个新增游戏化功能上线前都要回答：

- 它提升的是学习质量，还是只拉长在线时长？
- 是否会惩罚缺席、断签或休息？
- 是否会诱导深夜使用？
- 是否能被重复刷奖励？
- 老师或家长能否理解它的教育价值？
- 奖励来源是否可解释、可审计、有上限？
- 对 10-12 岁学生是否会造成不必要焦虑？
