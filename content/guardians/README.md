# 守护者 / Guardians

> 每一关都有一位守护者。守护者不是装饰——ta 是**这个算法的灵魂化身**。

## 守护者的三重身份

1. **出题人**：用 ta 自己的语气把题目讲给主角听
2. **通关恭喜者**：通过后给一段恭喜（带 ta 的性格）
3. **失败鼓励者**：失败时给一段鼓励（**永远不嘲讽**）

附加：每位守护者还有一道"跳级挑战题"，由 ta 出。

## 守护者性格 = 算法精神的拟人化

设计原则：守护者的性格应该和该关卡的算法/知识点呼应。例：

| 算法 | 适合的守护者特征 | 示例 |
|---|---|---|
| 输出 cout | 清晨的第一个发声者 | 一只爱说早安的麻雀 |
| 变量赋值 | 喜欢给东西贴标签 | 强迫症的小田鼠记账员 |
| 整除取模 | 严格按规矩切 | 按部就班的兔奶奶面包师 |
| if 分支 | 总在做选择 | 站在岔路口的猫头鹰 |
| for 循环 | 重复做同一件事不烦 | 每天数花瓣的蜗牛 |
| 嵌套循环 | 有耐心绣花的角色 | 织布的老蜘蛛 |
| 一维数组 | 喜欢列队 | 灯笼街上的萤火虫小队长 |

> 上面是举例，真正命名 + 设定由团队定，**不要让 AI 起名**——AI 起的名字会失去文化感。

## 设计建议（团队讨论时参考）

1. **章末守护者**（每章最后一关）应该比常规守护者更"重量级"——可能是被前面 11 个守护者推荐的"长辈"，或是章节地理的拟人（雾镇之灵）
2. **跨章呼应**：第 1 章某个守护者可以在第 2 章以背景形式再出现一次（"那只麻雀飞到了镜湖"）
3. **避免刻板印象**：不要全是"老者教授学生"的结构。可以有同龄玩伴守护者、害羞的守护者、调皮的守护者
4. **性别平衡**：15 位里建议至少 6 位女性气质角色
