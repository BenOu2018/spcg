# SPCG 小说正文草稿 V3：第16-18章

> 本组进入第6关：烈焰堆栈 vs 烬语，并开启“萤光数据海”剧情副本。  
> 本关对应 SPCG 6级，核心是栈、队列、循环队列、树、二叉树、哈夫曼、格雷码、DFS、BFS、一维 DP 和简单背包。

---

## 第16章 火牌入栈

烈焰堆栈的入口像一座倒扣的炉子。

轨车还没停稳，热浪就压了上来。犬虎从递归深渊带出的寒意瞬间被烧干，只剩胸口那块更硬的东西。

他没有和铁律说话。

铁律也没有解释。

星引坐在另一侧，脸色苍白。她说出真相后就一直沉默，像把自己也推进了某个边界条件。

广播响起。

“即将抵达第6关，烈焰堆栈。请挑战者确认隔热装备、手写记录和撤离路线。火场内允许观察无句建议，禁止自动生成通关代码。”

车门打开。

烬语站在火脉入口。

他身上有半边火焰纹身，火光从皮肤下缓慢流动，像随时会熄灭又不肯熄灭的余烬。他看见犬虎，没有摆出攻击姿态，只把一叠火牌放到石台上。

“第6关，烈焰堆栈。”

犬虎问：“你就是烬语？”

“是。”

“你为什么守在这里？”

烬语看了他一眼。

“先活过第一题，再问。”

第一题叫“火牌入栈”。

火牌一张张压进石槽，最后放入的必须最先取出。若取错顺序，火槽会喷出灰焰。

犬虎很快写出栈。

`push`，`pop`，`top`。

可题目加了撤销操作。每当维修队走错一步，就要撤回上一步状态。犬虎忽然想起递归深渊里的回溯，也想起自己现在最想撤回的，是星引说出那句话的瞬间。

他写错一次，把已经弹出的火牌又访问了一遍。

“空栈访问。”

烬语说：“火场里，不存在的东西不要伸手去拿。”

第二题“熔岩队列”让矿车按到达顺序过闸。先到先行，不能插队。犬虎以为这比栈简单，结果遇到多队入口合并时，忘了维护每辆车原来的到达时间。

烬语没有骂他。

“救援时，排队不是慢，是公平。”

循环队列更像火场本身。

环形火轨空间有限，车尾接近车头。犬虎必须用取模处理首尾移动，判断满和空。最开始，他把 `front == rear` 同时当作空和满，火轨系统直接报警。

星引终于开口。

“要留一个空位，或者额外记录数量。”

犬虎看向她。

她避开他的目光。

他按她说的改了。

题目通过。

火场深处升起一棵树。

不是植物，是由火脉和管道组成的结构。根在地下，枝向四周分叉，每个节点储存一段补给信息。

烬语说：“火树根脉。”

犬虎站在树前，忽然明白为什么第6关不只是栈和队列。火场不是一条线，救援也不是一条线。它有父子关系，有层级，有遍历顺序，也有压缩通信。

他先写树的基本结构，再写二叉树前序、中序、后序遍历。前序像先检查自己再下发命令；中序像按某种排序读取；后序像先处理子节点再回到根。

完全火树要求用数组下标定位父子节点。

左子是 `2*i`，右子是 `2*i+1`，父节点是 `i/2`。

犬虎写着写着，想起雾灯村粮仓里那些编号混乱的袋子。

如果当时有人教他这些，也许他不会总觉得自己笨。

夜里，火场暂时降温。

烬语把一杯苦得发焦的热茶递给犬虎。

犬虎没有接。

烬语说：“你在生谁的气？”

犬虎沉默。

“如果是生那个老头的气，就好好生。”烬语把茶放下，“别急着原谅。太快原谅的人，常常只是还没想明白。”

犬虎抬头看他。

烬语看着火树。

“火不会因为你理解它就不烫。但理解它，至少能让你知道该站在哪里。”

## 第17章 广搜撤离

第17章从哈夫曼火码开始。

火场通信带宽有限，频繁出现的信号应该用更短的编码。犬虎把每种信号的频率读入优先队列，反复取出频率最小的两个节点合并，构造哈夫曼树。

他写得很慢。

烬语却没有催。

“压缩不是为了省字。”他说，“是为了在火场里让重要信息更快到达。”

格雷码火门紧接着出现。

相邻门状态每次只能改变一位，否则火门会同时震动，引发连锁事故。犬虎第一次觉得那些看似奇怪的编码，有时是在现实里保命。

接下来是深搜火廊。

火廊像迷宫，但热浪会不断封路。DFS 要求沿一条路探到底，走不通再回退。犬虎用递归写，立刻想起深渊。他小心处理访问标记，避免在火廊里绕圈。

广搜撤离则完全不同。

火场里有许多被困点，题目要求从入口开始按层扩展，找出到每个点的最短撤离步数。犬虎用队列，把起点入队，再一层层推进。

他忽然理解 DFS 和 BFS 的差别。

DFS 像一个人拿着绳子深入黑暗。

BFS 像一队人举着灯，一圈一圈扩大安全区。

烬语看着他写完最短撤离路线，低声说：“我以前也这样教孩子。”

犬虎停下。

“你教过？”

“很多。”

星引接过话：“烬语以前走遍各域，给贫民孩子讲栈和队列。他说一个孩子只要能理解排队，就能理解公平；只要能理解撤销，就能理解犯错还能回来。”

烬语没有否认。

“后来呢？”犬虎问。

烬语的火焰纹身暗了一下。

“后来我遇见流萤。”

犬虎第一次听到这个名字。

火树寻路、余烬状态和背包蒜粒接连压下来。

火树寻路要求在二叉搜索树里定位补给节点。余烬状态是一维 DP：每走一段火线，记录到达该处时能剩下的最大余烬值。背包蒜粒则给出有限容量，要求选择最有效的补给组合。

犬虎在背包题里卡了很久。

他想全部带走。

蒜粒、冷却片、绷带、备用灯芯，每一样都像有用。

题目却只给固定容量。

他必须取舍。

烬语看着他。

“很讨厌吧？”

犬虎说：“嗯。”

“我也讨厌。”

烬语伸手按住胸口的火纹。

“救人时最讨厌的不是不知道怎么救，是知道每个人都该救，但你手里的东西只够救一部分。”

犬虎没有说话。

最终综合题“烈焰堆栈”启动。

他用栈处理撤销，用队列处理矿车和撤离，用树保存火脉结构，用 DFS 检查深处火廊，用 BFS 计算最短逃生路线，用 DP 和背包选择补给。

火脉一条条恢复。

系统提示：

“第6关，烈焰堆栈，通过。”

“称谓授予：栈火使者。”

烬语没有露出胜负表情。

他只拿出一枚半透明的水蓝色数据锚。

“替我去看看她。”

“流萤？”

烬语点头。

“告诉她，她不是任何人的负担。”

犬虎握住数据锚。

火场深处，水声响起。

## 第18章 萤海弱信号

萤光数据海不是主线关卡。

它没有排名，没有挑战席，也没有守卫宣告。数据锚打开后，犬虎只是从烈焰堆栈后方的一条降温管道滑入水下，像从火的心脏坠进一片安静的蓝。

水下城市遗迹悬在黑暗里。

废弃管道像道路，病历碎片像漂浮的纸鱼，远处有一个女孩坐在半塌的穹顶下。她身体局部半透明，微光从皮肤里透出来，像一只随时会消失的萤火。

“你是犬虎。”她说。

“你是流萤。”

流萤笑了一下。

“烬语又让别人来劝我吗？”

犬虎想起烬语的话。

“他说你不是负担。”

“他总这么说。”流萤低头看自己的手，“可系统不这么说。”

她把一段病历投影给犬虎。

无句诊断记录里，流萤体内许多微弱信号都被标成“噪声”。建议处理方式很简单：删除、平滑、忽略。

犬虎看见那些字，忽然想起寒霜冰桥上的缺失日志。

缺数据，不代表不存在。

弱信号，也不代表没意义。

萤海副本的第一段要求建立水下地图。犬虎把房间、管道和闸门建成节点与边。第二段用 DFS 追踪一条微弱数据流，第三段用 BFS 从入口扩展可达区域，第四段用 Flood Fill 标记连通水域。

这些能力在烈焰堆栈里刚练过。

可这里的情绪完全不同。

火场里是救援。

萤海里是倾听。

病历索引要求用哈希表快速定位记录。错误接口要求模拟旧协议读取，找出无句为什么把维生片剂量读错。实验日志要求从字符串碎片中拼出一段旧研究记录。

记录里写着：

“蒜粒并非天然资源。它是终端早期为加速文明学习而释放的能量介质。若由无句系统接管全部接口，人类将逐渐退出理解过程，底层纠错能力会快速退化。”

犬虎读到这里，背后发冷。

流萤轻声说：“所以我不是病人，对吗？”

犬虎不知道怎么回答。

她看着自己半透明的手。

“我是一条错误日志。”

犬虎摇头。

“你是人。”

流萤怔住。

犬虎把所有弱信号重新标记。他没有让系统删除那些异常点，而是把它们归类、索引、连通、生成样本。

手写板亮起：

“SPCG 新题胚生成：弱信号保留、错误接口复核、生命补给路径。”

“副本称谓授予：寻迹行者。”

流萤看着那行字，眼眶里浮起很浅的光。

“这些题，会有人做吗？”

“会。”犬虎说。

“如果他们觉得太难呢？”

犬虎想了想。

“那就有人把错法留下来，再变成下一批题。”

流萤第一次真正笑了。

离开萤光数据海前，她给了犬虎一枚数据碎片。

“这里面有星引当年的事故记录。”

犬虎手一紧。

“你知道？”

“萤海收很多被删掉的弱信号。”流萤说，“有些人以为删掉就不存在了。”

犬虎带着碎片回到烈焰堆栈外环。

铁律站在那里。

星引也在。

三个人之间隔着一段不长不短的距离，像一条还没写完的边。

犬虎没有质问。

他只是把数据碎片放进手写板。

屏幕亮起一行旧记录：

“星引源码感知力过载。若不封存，预计三日内神经系统崩解。”

星引脸色瞬间白了。

铁律闭了闭眼。

犬虎看着那行字，心里更乱了。

铁律没有完全说谎。

星引也没有无理取闹。

一个人活下来，另一个人失去自己。这样的选择，程序里没有简单的 `true` 或 `false`。

远处，第7关磐石要塞的轨车灯亮起。

犬虎背起手写板，带上寒霜日志、深渊称谓、烬语数据锚和流萤弱信号。

他终于明白，自己不是一关一关打败守卫。

他是在一层一层读懂他们留下的错误、隐瞒和救人方式。

轨车启动。

磐石要塞在黑暗里越来越近。
