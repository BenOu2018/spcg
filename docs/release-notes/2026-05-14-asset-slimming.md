# 2026-05-14 Public Asset Slimming

## Summary

This release stops exposing the full `assets` and `assets/video` trees through Next.js public routes. Source files remain in the repository/workspace for future regeneration, but only a generated runtime subset is copied into `apps/web/public/assets`.

Production default:

- `/assets` contains only runtime assets generated by `npm run prepare:public-assets`.
- `/video` is not generated unless `SPCG_SOLUTION_VIDEO_ENABLED=true`.
- Student-facing level data hides `solutionVideoUrl` while the video flag is disabled.

## Public Exposure Change

Before slimming, `apps/web/public/assets` and `apps/web/public/video` were symlinks to the full source trees. The effective public exposure was:

## Release Commands

Run before build/deploy:

```bash
npm run prepare:public-assets
npm run assets:audit
```

`npm run web:dev` and `npm run web:build` now run `prepare:public-assets` automatically.

To temporarily restore solution videos in a future release, set:

```bash
SPCG_SOLUTION_VIDEO_ENABLED=true
```

Then rerun `npm run prepare:public-assets` so `/video` is generated again. Leave the flag unset or set to any value other than `true` to keep videos hidden and unpublished.
