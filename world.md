# 游戏世界观入口

> 当前阶段：记录新的游戏世界观与系统体验重塑方向。
> 第一轮开发：只保留 `/world` 作为算法遗迹主页，进入后直接到原 `/map`。
> 不再额外维护 `/world/map` 或 `/world/level/[id]` 平行路线。

## 第一轮开发总纲

第一轮不重开一个 web，也不复制一套 OJ。

新的入口方式是：

```txt
/world -> /map -> /level/[id]
```

`/world` 负责建立世界气质，`/map` 仍然是正式进入 OJ 系统的地图。

这意味着：

- 不再做额外一层 world map。
- 不再做额外一层 world level。
- 不再把同一道题包装成两套不同页面。
- 原 `/map` 继续负责关卡、进度、解锁、跳转。
- 原 `/level/[id]` 继续负责题面、编程 UI、判题和提交记录。
- `/world` 首页的所有“进入遗迹 / 遗迹地图 / 进入地图”入口都指向 `/map`。

第一轮要做的是：

```txt
只保留 /world 入口页。
```

现阶段不在原 `/map` 上叠加 world 分支功能。
地图、关卡、判题和编程 UI 全部保持原样。

### 第一轮复用策略

| 模块 | 第一轮处理 |
|---|---|
| 题库数据 | 复用原 `level` 数据，不改数据库原题 |
| 判题系统 | 复用原 `runCodeAction`、`submitCodeAction`、远程判题与进度写入 |
| 进度系统 | 复用原 `progressRecords` |
| 地图 | 保持原 `/map` 和 `LevelMap`，不叠加 world 层 |
| 编程 UI | 复用原 `/level/[id]` |
| world 首页 | 只做品牌入口和情绪建立 |
| map 增强 | 本轮不做，避免影响原 OJ 地图体验 |

## 核心重塑点

### 世界
核心气质已经确定：

- 神秘
- 孤独
- 文明残骸
- 算法像古代失落知识

玩家不再被提示“来刷题”，而是进入一座失落文明留下的遗迹。算法不是课程表里的知识点，而是被封存在废墟中的古代思维技术。每一次操作都应该像是在解读、修复、唤醒一个已经沉睡太久的系统。

#### 开场画面

黑暗空间中，大量残缺的代码碎片漂浮：

```txt
if
for
while
return
else
int
loop
dfs
bfs
dp
graph
memory
```

远处有巨大残破石碑。

石碑上闪烁一句话：

> 人类最后一次独立思考，已无法追溯。

随后，一个古老系统苏醒：

```txt
正在检测思维活性……
检测失败。
发现低级逻辑残留。
是否进入修复协议？
```

按钮：

```txt
进入遗迹
```

这里已经不是“登录 / 注册 / 开始学习”，而是“进入遗迹”。用户情绪会完全不同：不再觉得自己来刷题，而是进入了一座失落文明。

#### 名词体系

| 普通 OJ | 世界名词 |
|---|---|
| 题目 | 遗迹 |
| 提交 | 解译 |
| AC | 修复成功 |
| WA | 逻辑断裂 |
| Runtime Error | 遗迹崩塌 |
| 排名 | 思维波段 |
| 等级 | 思维阶位 |
| 题解 | 古代记录 |
| AI分析 | 文明回响 |

这一步极其关键：用户每操作一次，都会被世界观强化一次。

#### 首页 V2 视觉方向

首页从“单屏苏醒入口”升级为“游戏官网入口”：

- 主标题：算法遗迹
- 品牌呈现：SPCG 算法遗迹
- 主按钮：获取遗迹通行证
- 底部按钮：立即获取通行证
- 视觉关键词：古老遗迹、金色铭文、青绿色数据雨、石质界面、遗迹地图、探索者身份

首页模块：

- 顶部导航：遗迹入口、挑战榜、遗迹档案、探索者、关于我们、探索者登录。
- Hero 区：巨大遗迹门、探索者背影、算法机关、公告面板。
- 遗迹地图：第一遗迹已开放，后续遗迹以未解锁状态呈现。
- 能力收益：算法思维、实战能力、AI 分析、专属称号、探索乐趣。
- 体系对应：LV1-LV4 与更高遗迹，用“思维阶位”感替代普通等级感。

所有关键中文标题、按钮、公告、节点名都由页面文本渲染，不烘进图片，以保证世界名词准确。

### 反馈
第二核心已经确定：

用户不是在提交代码。
用户是在激活遗迹。

真正让人沉浸的不是奖励，而是世界对玩家行为产生回应。只要用户感觉“这个世界在观察我、回应我”，沉浸感就会迅速形成。

普通 OJ 的反馈链路太短：

```txt
写代码 -> 提交 -> 等待 -> AC / WA -> 结束
```

这个世界必须把每一次代码行为改造成一次遗迹响应：

```txt
解译逻辑 -> 遗迹解析 -> 古代结构响应 -> 世界局部被修复
```

#### 第一层反馈：即时响应

提交按钮不使用“提交”或“运行”。

默认文案：

```txt
开始解译
```

更强仪式感场景可使用：

```txt
激活遗迹
```

提交后的等待状态：

```txt
遗迹正在解析你的逻辑……
```

编译成功：

```txt
古代结构已响应。
```

修复成功是最重要的高潮，不只显示 Accepted，而是让用户看见世界发生变化：

- 黑暗遗迹亮起蓝色纹路
- 石碑开始发光
- 尘封机械重新启动
- 地图亮起一块区域

成功文字：

```txt
修复成功。
失落逻辑已恢复。
```

这里用户获得的不是“做对题”的感觉，而是“修复世界”的感觉。这是普通刷题平台和游戏化遗迹世界的核心差别。

#### 第二层反馈：连续修复

连续修复反馈不以金币、积分为主，而以情绪回应为主。

连续修复 3 次：

```txt
思维同步率提升。
```

连续修复 10 次：

```txt
你开始听见遗迹回响。
```

深夜连续解译：

```txt
检测到长时间思维活跃。
旧文明正在注视你。
```

这类反馈的目标不是奖励用户，而是让用户感到自己的状态被世界捕捉到了。系统像一个沉睡文明的接口，正在因为玩家持续思考而逐渐苏醒。

#### 第三层反馈：失败也要回应

失败不应该把用户推回“我菜”的情绪里。
失败应该表达为“遗迹尚未解开”。

WA：

```txt
遗迹拒绝了当前逻辑。
```

TLE：

```txt
该结构过于冗长。
远古核心停止响应。
```

RE：

```txt
遗迹发生崩塌。
检测到危险逻辑。
```

这样失败不再只是打击，而是破解过程的一部分。用户面对的不是一道判错自己的题，而是一座还没有被完全理解的古代结构。

反馈系统的最终目标：

```txt
不是告诉用户“你做对了 / 你做错了”。
而是让用户感到：
这个世界正在因为我的思考而变化。
```

### 身份
核心原则：

平台不是在培养“刷题用户”。
平台是在培养“遗迹中的某种人”。

人不会长期坚持“做题”，但会长期维护自己的角色、称号、世界身份，以及自己在世界里的存在感。这个世界里的玩家不是学生、程序员或勇者，而是失落思维文明的继承者。

#### 初始身份

新进入玩家的身份：

```txt
余烬
```

含义：

```txt
文明熄灭后，最后残留的思维火种。
```

这不是一个强者称号，而是一个带有孤独感和可能性的起点。玩家刚进入遗迹时，还没有被文明承认，只是被检测到仍然保留微弱的独立思考能力。

第一次修复成功后，身份变化为：

```txt
觉醒者
```

因为玩家第一次真正独立解决问题，证明自己不只是残留火种，而是仍然可以回应古代逻辑的人。

#### 成长阶位

身份阶位不使用青铜、白银、王者这类竞技化命名，而使用神秘文明感更强的名称。

| 阶位 | 身份名称 | 含义 |
|---|---|---|
| Lv1 | 余烬 | 文明熄灭后残留的思维火种 |
| Lv2 | 觉醒者 | 第一次独立修复遗迹的人 |
| Lv3 | 解译者 | 能稳定读取遗迹逻辑的人 |
| Lv4 | 遗迹行者 | 开始穿行于多个遗迹区域的人 |
| Lv5 | 逻辑编织者 | 能把分散规则重新组织成结构的人 |
| Lv6 | 深层观测者 | 能看见表层题意之下隐藏结构的人 |
| Lv7 | 文明修复者 | 不只破解单点遗迹，而是在修复文明系统的人 |
| Lv8 | 远古思维体 | 思维模式开始接近旧文明核心的人 |

身份不是简单等级。
身份表达的是：世界怎么看你。

普通 OJ 显示：

```txt
等级：12
```

这个世界显示：

```txt
身份：遗迹解译者
当前思维同步率：17%
已修复遗迹：23座
```

这会让玩家感觉自己不是在完成任务列表，而是在世界里逐渐形成一种可被识别的存在。

#### 稀有身份

身份必须具有稀有性。

有些称号不面向所有人公开，也不需要提前列出完整条件。未知会产生探索欲，玩家会开始猜测世界还记录了哪些行为。

全球稀有身份示例：

```txt
递归深潜者
```

获得条件：

```txt
连续不用题解通过 50 道递归题。
```

隐藏身份示例：

```txt
沉默观测者
```

获得条件：

```txt
凌晨 3 点后连续活跃 7 天。
```

奇异身份示例：

```txt
被遗迹记录之人
```

获得条件：

```txt
未知。
```

不要公开全部称号。公开一部分，让玩家知道系统存在；隐藏一部分，让玩家相信世界比界面显示出来的更深。

#### 身份需要被看见

身份如果只藏在个人资料页里，就不会形成长期牵引。

身份必须出现在玩家被他人看见的地方：

- 评论区
- 排行榜
- 题解作者信息
- 个人主页
- 遗迹修复记录
- 小队或社群列表

评论展示不只显示用户名：

```txt
[遗迹行者]
BenOu
```

排行榜不只显示 Rank：

```txt
第三观测席位
```

用户需要不断看到：自己在这个世界中被如何命名、如何记录、如何承认。

#### 算法风格身份

身份要和算法风格绑定。

这是未来最强的长期系统之一，因为它会把算法偏好变成玩家人格。

喜欢 DFS 的玩家，逐渐获得：

```txt
深渊探索者
```

喜欢 DP 的玩家，逐渐获得：

```txt
未来推演者
```

喜欢图论的玩家，逐渐获得：

```txt
路径观测员
```

经常优化复杂度的玩家，逐渐获得：

```txt
时间压缩者
```

这类身份不只是奖励，而是在告诉玩家：

```txt
你的解题方式已经被世界识别。
```

当用户看到某个称号时，产生的不是“我拿到了一个徽章”，而是“这就是我”。

身份系统的最终目标：

```txt
不是让用户变成更高等级。
而是让用户越来越相信：
我在这个遗迹文明中，有一个被记录、被看见、被承认的位置。
```

### 情绪
核心原则：

绝大多数 OJ 只有功能，没有情绪。
但真正让人记住的游戏，本质上都是情绪体验。

用户不会长期记得自己做了哪道题、学了哪个知识点，但会记得：

```txt
那个夜晚，我第一次点亮遗迹。
```

这个世界的目标不是让用户“学”，而是让用户产生感觉。学习只是行为结果，情绪才是长期记忆。

最适合这个世界的情绪方向：

- 孤独感
- 神秘感
- 文明回响感

因为这里不是热闹的竞技场，而是一座被 AI 时代遗忘的算法遗迹。用户不是冲进关卡刷奖励，而是在安静的废墟里重新点燃独立思考。

#### 背景情绪

不要做成：

- 热血页游
- 炫酷科技站
- 儿童编程 UI
- 到处跳动的奖励中心

真正适合的是：

```txt
安静的遗迹感。
```

画面与氛围应持续靠近这些元素：

- 黑暗背景
- 微弱蓝光
- 缓慢漂浮的代码碎片
- 风声
- 远古机械低鸣
- 偶尔闪烁的旧屏幕
- 像断电很久后重新接入的系统界面

用户进入页面时，应该感觉自己不是打开一个学习网站，而是一个人走进文明废墟。

背景情绪的目标：

```txt
让用户安静下来。
让用户愿意思考。
让用户感觉这里比普通网页更深。
```

#### 深夜情绪

这个产品天然适合夜晚使用。

深夜刷题不应该只是白天界面的延续，而应该成为特殊情绪场景。

凌晨后，系统可以逐渐进入低频状态：

- 背景更暗
- 环境音更安静
- 光效更慢
- 提示文案更克制
- 在线状态更像文明扫描结果

深夜状态文案：

```txt
遗迹进入低频状态……
```

或者：

```txt
当前在线思维体：17
```

这里不说“在线人数”，因为那只是统计。
要表达的是：深夜仍然有人在思考。

深夜连续活跃时，系统可以回应：

```txt
你似乎比多数人更接近旧时代。
```

这类情绪不需要大声。它的力量在于让用户觉得：此刻世界正在安静地看见我。

#### 修复成功的情绪释放

AC 不应该做成爆炸、金币、彩带。

这里真正适合的是：

```txt
文明重新苏醒。
```

修复成功时的反馈应该克制、缓慢、带有仪式感：

- 石碑纹路缓慢亮起
- 微弱低频共鸣出现
- 远处建筑恢复供能
- 黑暗区域出现一小块光
- 地图上某个断裂节点重新连接

成功文案：

```txt
检测到有效逻辑。
遗迹恢复运行。
```

这不是爽游式高潮，而是克制型高潮。

它的情绪不是“赢了”，而是：

```txt
我让一个沉睡很久的东西重新回应了。
```

#### 失败情绪

普通平台把失败做成打击。
这个世界必须把失败做成探索受阻。

连续 WA 时，不显示冷冰冰的 Wrong Answer，而是：

```txt
遗迹结构仍存在冲突。
```

连续失败很多次时，系统不要嘲讽，也不要用过度鼓励打断氛围。

可以显示：

```txt
旧文明留下的结构，
本就不是轻易能理解的。
```

这句话会改变用户对失败的解释。

用户感受到的不是：

```txt
我太菜了。
```

而是：

```txt
这个遗迹真的难。
我还在接近它。
```

失败情绪的目标不是让失败消失，而是让失败变成探索的一部分。

#### 系统记住玩家情绪

真正高级的沉浸感来自：

```txt
系统像是记得我。
```

很久没上线后再次登录：

```txt
遗迹沉寂了很久。
欢迎回来，解译者。
```

某题卡了三天后终于修复成功：

```txt
该遗迹已等待你很久。
```

长时间深夜活跃：

```txt
你似乎比多数人更接近旧时代。
```

用户不应该觉得系统只是记录数据，而应该觉得这个世界正在观察自己的状态、节奏和执念。

这种“被世界记住”的感觉，会比金币、积分、连胜更深。

#### 情绪边界

这个世界千万不要故意热闹。

核心魅力不是爽游，而是：

```txt
安静。
孤独。
思考。
文明余烬。
```

这是一条少见的路线，但一旦做出来，会形成非常强的气质。

情绪系统的最终目标：

```txt
不是让用户觉得这里很好玩。
而是让用户在某个安静的时刻觉得：
我正在和一个失落文明发生连接。
```

### 引导
核心原则：

不要让用户感觉：

```txt
系统在教我。
```

而要让用户感觉：

```txt
我在探索。
```

这是引导系统的根本区别。

绝大多数学习平台的引导都是教程、提示、下一步、新手指引。用户一看到这些词，就会立刻意识到“我要开始上课了”，沉浸感会消失。

这个世界的引导必须隐藏在遗迹探索中。用户不是被带着学，而是在不断遇到问题、观察线索、尝试修复，最后自己理解。

#### 不要叫教程

命名必须彻底从教学语言切换到世界语言。

| 普通平台 | 世界名词 |
|---|---|
| 新手教程 | 初始遗迹 |
| 第一课 | 第一座遗迹 |
| 学习路径 | 文明路径 |
| 知识点 | 遗迹结构 |
| 提示 | 古代记录 |
| AI讲解 | 文明回响 |

用户不应该觉得自己在进入课程，而应该觉得自己在进入一段文明路径。

#### 用问题引导，而不是用知识引导

普通教学会说：

```txt
今天学习 for 循环。
```

这太像课堂。

这个世界应该先给出一个需要被修复的问题。

例如第一遗迹：

```txt
《失控计数塔》
```

描述：

```txt
旧时代用于统计资源的塔楼已经停止运行。
你需要恢复它的计数结构。
```

用户进入后，才发现本质是在理解循环。

这一步极其重要。

用户不是：

```txt
为了学循环而学循环。
```

而是：

```txt
为了修复遗迹，必须理解它。
```

这就是游戏化学习真正的核心。

#### 不要一次解释太多

沉浸式引导应该先让用户行动，而不是先讲语法。

不要一开始弹出：

```txt
for 循环语法：
for(int i=0;i<n;i++)
```

而是先让用户看到遗迹现象：

```txt
石碑上出现了一组不断变化的数字。
似乎存在某种重复规律。
```

用户会自然开始观察、尝试、猜测。

当用户已经产生问题后，再给出古代记录：

```txt
重复结构，是旧文明最基础的逻辑之一。
```

之后再逐渐出现具体形式：

```txt
for (...)
```

这不是老师讲课，而是遗迹给线索。

引导节奏应该是：

```txt
先现象。
再疑问。
再线索。
再结构。
最后命名。
```

#### 引导必须有发现感

知识应该像是从世界中被发现的，而不是从老师那里被灌输的。

例如用户第一次接触 DFS，不要直接说：

```txt
深度优先搜索（DFS）是一种……
```

而是让用户进入：

```txt
《裂谷回声遗迹》
```

用户在遗迹中发现：

- 有很多分岔道路
- 一条路需要走到底
- 走不通后必须返回
- 路径选择带有回溯感

这时系统才提示：

```txt
旧文明似乎使用某种“深入后返回”的探索逻辑。
```

然后用户再接触 DFS。

这样形成的体验是：

```txt
知识来自世界。
```

而不是：

```txt
知识来自老师。
```

#### AI 不要像老师

AI 在这个世界里不能直接像老师一样下结论。

最忌讳的表达：

```txt
这里你应该使用 DFS。
```

这种话会瞬间破坏沉浸感。

AI 应该像遗迹中的古代辅助系统，提供线索，而不是给答案。

更合适的表达：

```txt
检测到结构存在深层分支。
也许，
你需要一种“先深入，再返回”的方式。
```

它不直接说 DFS，而是引导用户思考。

AI 的语气应该遵循：

- 少下结论
- 多描述结构
- 少说答案
- 多给线索
- 少像老师
- 多像旧文明接口

#### 用户必须自己悟到

真正会上瘾的学习，不是“被教会”，而是：

```txt
我自己想明白了。
```

系统要大量设计这种路径：

- 观察
- 猜测
- 尝试
- 破解
- 顿悟

而不是：

- 听课
- 记忆
- 背模板

引导系统的任务不是把知识直接送到用户面前，而是创造一种环境，让用户觉得自己正在把知识从遗迹里挖出来。

#### 引导系统的定位

现在这个项目已经不是普通的游戏化 OJ。

它更接近：

```txt
文明探索型学习世界。
```

普通 gamification 是把积分、徽章、等级贴到学习上。

这个世界要做的是：

```txt
让知识变成遗迹。
让问题变成入口。
让答案变成修复。
让理解变成发现。
```

引导系统的最终目标：

```txt
不是让用户按下一步走完教程。
而是让用户在探索中自然学会：
我为什么需要这个结构。
```

### 仪式感
核心原则：

仪式感不是花里胡哨，不是动画很多，也不是特效很炸。

真正高级的仪式感是：

```txt
某个瞬间被认真对待。
```

用户会忘记功能，但不会忘记：

- 第一次进入世界
- 第一次点亮遗迹
- 第一次获得身份
- 第一次听见系统回应自己
- 第一次感到世界记住了自己

这些瞬间必须被放大，因为它们会成为用户对这个世界的记忆锚点。

#### 第一次进入世界

普通平台的流程是：

```txt
注册 -> 登录 -> 进入主页
```

没有情绪，也没有记忆。

这个世界第一次进入时，不应该直接进入主页，而应该先经历：

```txt
遗迹启动仪式。
```

画面可以从黑暗开始。

缓慢出现：

```txt
系统沉寂：
2149 天
```

随后：

```txt
检测到新的思维波动……
```

接着：

```txt
正在建立同步……
```

最后：

```txt
欢迎进入：
算法遗迹
```

然后用户才真正进入世界。

这一刻的目标不是解释功能，而是让用户意识到：

```txt
我不是打开了一个网站。
我是进入了一个沉睡很久的地方。
```

#### 第一次修复成功

第一次 AC 是整个产品最关键的高潮之一。

绝大多数 OJ 把它浪费成了一个普通状态：

```txt
Accepted
```

这个世界必须把第一次成功做成：

```txt
第一次点亮文明。
```

用户第一次修复成功后，不直接弹结果。

流程可以是：

- 环境短暂安静
- 远处遗迹亮起
- 低沉共鸣声出现
- 石碑显现文字

石碑文字：

```txt
检测到完整逻辑。
旧文明恢复响应。
```

随后身份更新：

```txt
余烬 -> 觉醒者
```

这一刻必须被认真对待，因为用户会记住：

```txt
我第一次让这个世界亮了起来。
```

#### 获得称号

称号不是普通奖励。

不要显示：

```txt
获得称号：DFS高手
```

这太像后台成就系统。

更适合的表达：

```txt
遗迹记录了你的探索方式。
```

随后：

```txt
称号觉醒：
「深渊探索者」
```

称号获得不是“发奖”，而是“世界识别了你”。

用户感受到的应该是：

```txt
我的行为被这个文明记录了。
```

#### 深夜仪式

凌晨后的状态应该成为特殊仪式。

例如凌晨 2 点后，整个 UI 进入：

```txt
遗迹低频模式。
```

表现方式：

- 背景更暗
- 音乐更空灵
- 光效更慢
- 提示更少
- 文案更像系统低频广播

可能出现：

```txt
仍有少量思维体保持活跃……
```

或者：

```txt
深夜状态已同步。
```

这不是夜间模式那么简单。

它强化的是：

```txt
孤独思考者联盟。
```

深夜仍在解译的人，会觉得自己和少数仍然清醒的思维体共同存在于遗迹中。

#### 长时间坚持后的仪式

连续学习不应该叫签到。

不要显示：

```txt
连续签到 30 天。
```

更适合的是把坚持表达成世界痕迹。

连续探索 30 天时：

- 黑暗大厅出现
- 中央石碑完全点亮
- 旧文明纹路从地面向外扩散

文字：

```txt
你已经在遗迹中留下了持续回响。
```

随后：

```txt
文明认可度提升。
```

这里奖励不是重点。

重点是让用户觉得：

```txt
我的坚持在世界中留下了痕迹。
```

#### 世界记得你

最强的仪式感来自：

```txt
世界记得我。
```

某个用户卡题很久后终于通过：

```txt
该遗迹已等待你许久。
```

很久没上线后回来：

```txt
部分遗迹仍保持着你上次离开的状态。
```

用户成为高阶玩家后：

```txt
越来越多后来者，
正在阅读你留下的记录。
```

这一刻，用户开始相信：

```txt
我真的存在于这个世界。
```

这比任何积分、金币、经验值都更深。

#### 仪式感的意义

真正的仪式感不是奖励，而是意义。

普通 OJ：

```txt
AC +100 经验
```

结束。

这个世界：

```txt
又一段失落逻辑，
重新回到了人类文明。
```

前者是数值。
后者是意义。

仪式感系统的最终目标：

```txt
不是让用户觉得自己被奖励了。
而是让用户觉得：
我的行为在这个世界中有意义。
```

## 世界观闭环

现在，六个核心已经串起来：

- 世界：算法是古代失落知识
- 反馈：用户不是提交代码，而是激活遗迹
- 身份：用户不是刷题者，而是思维文明的继承者
- 情绪：核心气质是安静、孤独、思考、文明余烬
- 引导：系统不直接教学，而是让用户探索和顿悟
- 仪式感：关键瞬间被认真对待，行为获得意义

这个项目已经不是普通 OJ。

它真正要成为：

```txt
一个关于人类重新夺回思考能力的文明遗迹世界。
```

## 工作方式

- 本文件作为独立世界观入口。
- 后续围绕六个关键点逐步展开。
- 在明确需要前，不改动原有题目、题单和系统结构。
