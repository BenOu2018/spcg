## 1. 核心种子提示词（雪花第1层）

```
作为专业作家，请用"雪花写作法"第一步构建故事核心：
主题：犬虎——一个刚入世无知的少年，在编程与算法的世界中不断闯关成长，历经磨难最终成为一代宗师
类型：科幻成长冒险
篇幅：约27章（每章3000字）

背景设定：
- 故事发生在"算法大陆"，一个由代码逻辑构建的科幻世界
- 世界运转依赖两种资源：金币（通用货币）和蒜粒（类似《沙丘》香料的稀缺能源，驱动一切高级科技和生命维持系统）
- 蒜粒正在不可逆地衰变，大陆面临终极危机
- 主角犬虎必须通过9道关卡（每关有一名守卫阻拦），从无知少年成长为拯救大陆的一代宗师

请用单句公式概括故事本质，例如：
"当[主角]遭遇[核心事件]，必须[关键行动]，否则[灾难后果]；与此同时，[隐藏的更大危机]正在发酵。"

要求：
1. 必须包含显性冲突（闯关成长）与潜在危机（蒜粒衰变 / 守卫背后的阴谋）
2. 体现犬虎的核心驱动力（从无知到觉醒）
3. 暗示世界观关键矛盾（蒜粒资源争夺、守卫的真实身份）
4. 使用25-100字精准表达

仅返回故事核心文本，不要解释任何内容。
```

## 3. 世界构建矩阵提示词（三维度交织法）

```
基于以下元素：
- 内容指导：SPCG 编程闯关游戏世界观，9关制，经济体系为金币+蒜粒
- 核心冲突："{core_seed}"

为服务上述内容，请构建三维交织的世界观：

1. 物理维度：
- 空间结构：算法大陆的9大区域（对应9道关卡），每个区域的地理特征映射一种编程/算法概念
- 蒜粒生态：蒜粒的产地、提炼方式、衰变规律、对科技和生命的影响
- 科技体系：基于蒜粒驱动的科技等级（从基础编译到高级AI），科技与蒜粒消耗的矛盾
- 时间轴：大陆的关键历史事件——蒜粒的发现、第一次蒜粒战争、守卫制度的建立

2. 社会维度：
- 权力结构：谁控制蒜粒，谁就控制大陆——蒜粒议会 vs 自由编码者联盟
- 金币经济：普通交易用金币，高级科技和关卡通行需要蒜粒，形成两级经济体系
- 守卫制度：守卫的选拔、权力、义务，以及守卫内部的派系分裂
- 文化禁忌：禁止私自合成蒜粒（类似沙丘中禁止人工制造香料）

3. 隐喻维度：
- 蒜粒 = 知识/智慧的隐喻（越稀缺越珍贵，获取需要付出代价）
- 9道关卡 = 学习成长的9个阶段（从基础到精通）
- 守卫 = 成长路上的障碍与考验（部分守卫本身也是被困住的人）
- 贯穿全书的视觉符号：源码碎片（散落在各关卡的古老代码残片，拼合后揭示大陆真相）

要求：
每个维度至少包含3个可与角色决策产生互动的动态元素。
仅给出最终文本，不要解释任何内容。
```

```

- 内容指导：SPCG 游戏小说，27章，9关结构（每关约3章：铺垫→对决→收获）
- 小说架构：
{novel_architecture}

设计27章的节奏分布：
1. 章节集群划分：
- 每3章构成一个关卡单元（第1章铺垫+第2章对决+第3章收获/过渡）
- 关卡之间设置"认知过山车"（紧张→缓冲→更大的紧张）

2. 每章需明确：
- 对应关卡编号和守卫
- 章节定位（角色/事件/主题等）
- 核心悬念类型（信息差/道德困境/时间压力/科技谜题等）

- 伏笔操作（埋设/强化/回收）
- 认知颠覆强度（1-5级）

输出格式示例：
第n章 - [标题]
对应关卡：第X关 / 守卫：[守卫名]
本章定位：[角色/事件/主题/...]
核心作用：[推进/转折/揭示/...]
悬念密度：[紧凑/渐进/爆发/...]
伏笔操作：埋设(A线索)→强化(B矛盾)...
认知颠覆：★☆☆☆☆
蒜粒衰变：[XX%剩余]
本章简述：[一句话概括]

要求：
- 使用精炼语言描述，每章字数控制在100字以内
- 合理安排节奏，确保悬念曲线连贯
- 蒜粒衰变从100%递减到危险值，制造全局紧迫感
- 在第27章前不要出现完全的大结局

仅给出最终文本，不要解释任何内容。
```

## 6. 章节正文写作提示词（第一章 / 后续章节）

沿用 AI_NovelGenerator 原版 `first_chapter_draft_prompt` 和 `next_chapter_draft_prompt`，
在 `user_guidance` 字段中注入以下 SPCG 专属指导：

```
SPCG 专属写作指导：
1. 蒜粒描写规范：蒜粒呈半透明琥珀色，散发微弱脉冲光芒，触碰时有温热感和低频嗡鸣。高纯度蒜粒可驱动城市级科技设施，低纯度蒜粒用于日常能量补给。蒜粒衰变时颜色变暗、脉冲减弱直至碎裂成灰。
2. 金币描写规范：六边形金属币，边缘刻有算法纹路，可在大陆任何交易所通用。
3. 战斗/对决场景：守卫与犬虎的对决以"编程挑战"为核心形式，但需用武侠/科幻化的叙事手法呈现（如"代码如剑气般展开""算法链条缠绕而来"）。
4. 成长体现：每过一关，犬虎的"源码感知力"提升一级，外在表现为瞳孔中出现更多代码流光。
5. 游戏衔接：每章结尾预留一个可转化为游戏内对话或选择分支的"决策点"。
6. 语言风格：面向青少年读者，热血但不幼稚，科幻但不晦涩。
```

## 使用流程

按 AI_NovelGenerator 的标准流程执行：

1. **核心种子** → 运行第1节提示词 → 输出 `core_seed.txt`
2. **角色动力学** → 运行第2节提示词 → 输出 `character_dynamics.txt` + `character_state.txt`
3. **世界构建** → 运行第3节提示词 → 输出 `world_building.txt`
4. **情节架构** → 运行第4节提示词 → 输出 `Novel_architecture.txt`
5. **章节目录** → 运行第5节提示词 → 输出 `Novel_directory.txt`
6. **逐章写作** → 运行第6节提示词 → 输出各章正文
7. **每章完成后** → 更新前文摘要 + 角色状态表
