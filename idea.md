# 想法记录簿 / Ideas Backlog

> 开发过程中浮现的功能/改进想法，先记下来不立即决策。
> 每周五回顾一次，把成熟的搬到对应版本路线图。

## 用法

1. **任何时候有想法都加一行**——不要在脑子里憋
2. **不评估、不删除、不立即实现**——记下就走
3. **每周五同步会回顾**——决定"立即做 / v0.2 / v0.3 / 不做"
4. **被采纳的想法搬到对应 plan.md**——本表只标"已采纳"

---

## 类型分类

- **内容**：故事 / 题目 / 角色 / 文案
- **机制**：游戏化规则 / 激励 / 教学方法
- **体验**：动效 / 视觉 / 音效 / 交互
- **工程**：架构 / 性能 / 安全 / 工具
- **商业**：付费 / 合作 / 营销 / 数据
